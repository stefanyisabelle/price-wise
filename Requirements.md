Devem ser utilizados os princípios de componentização para dividir sua aplicação em 3 páginas em componentes reutilizávei com pelo menos 4 componentes
Criar um repository que com respostas simualadas do servidor
Realizar o uso correto de estados, props e hooks (como useState, useEffect).
A aplicação deve implementar a navegação entre páginas utilizando o React Router
Além disso, devem ser utilizados pelo menos três hooks (ou funções equivalentes)
relacionados à navegação, como:
● useNavigate (ex: redirecionamento de rotas);
● useParams (ex: captura de parâmetros da URL);
● useLocation (ex: leitura da URL atual).
A aplicação deve incluir uma rota de erro para URLs inexistentes (ex: exibir uma página
404 not found).
Incluir alguns elementos para melhorar a experiência do usuário, tais como:
● Exibir feedback visual após ações do usuário, como: cliques, envios de
formulário, carregamento de dados;
● Tooltips explicativas em botões ou ícones;
● Mensagens condicionais contextuais, como “nenhum item encontrado” ou “erro
ao carregar dados”.
● Indicadores de carregamento, sucesso ou erro;
Desenvolver um layout adaptável, que se ajuste adequadamente a diferentes tamanhos
de tela (desktop, tablet e celular). Não é necessário que o design seja perfeito em todos
os dispositivos, mas o conteúdo deve permanecer legível e funcional.
Organização e documentação dos códigos
Organização clara da estrutura de pastas e arquivos no projeto bem como o uso adequado de convenções de nomenclatura e boas práticas (nomes de
componentes, arquivos, etc.)
